🎯 What is OMNITECH?

OMNITECH OMEGA is an autonomous AI quantization optimizer that uses Reinforcement Learning (UCB1 bandit) to automatically find the optimal GGUF quantization level for Large Language Models. Instead of manually testing each quant, it learns which quantization delivers the best tradeoff between speed (TPS), accuracy, and model size.

🏗️ System Architecture
🧠

RL Quantizer (UCB1)

core/quantizer_rl.py

The brain of OMNITECH. Uses Upper Confidence Bound 1 bandit algorithm to select quantization levels, dispatch jobs, collect rewards, and converge on the optimal GGUF quant automatically.

UCB1 Bandit Reward Loop llama.cpp
📋

UCB1 Policy

core/policy.py

Persistent policy engine with exponential moving average score updates. Forces exploration of untried quants first, then uses UCB1 formula to balance explore/exploit. State saved to JSON.

EMA Updates Persistent State
⚙️

Worker (Multi-GPU)

core/worker.py

Consumes benchmark jobs from Redis queue. Auto-detects NVIDIA GPUs via nvidia-smi. Runs llama.cpp across all available GPUs, aggregates TPS, and pushes results back to Redis.

Multi-GPU CUDA Redis Queue
🔄

Scheduler

core/scheduler.py

Job queue rate limiter. Monitors Redis queue depth and sets a "scheduler_ready" gate flag. Prevents flooding by pausing the RL loop when queue exceeds MAX_QUEUE_SIZE (5 jobs). Heartbeat monitored.

Rate Limiter Heartbeat
🛡️

Security AI

security/security_ai.py

Anomaly detection layer that monitors for job flooding (queue > 50), abnormal TPS spikes (> 500 t/s), reward manipulation, and missing worker/scheduler heartbeats. Alerts logged to Redis + file.

Anomaly Detection Alert Engine
🌐

API + WebSocket

api/app.py · api/stream_server.py

Flask REST API serving live dashboard HTML with Chart.js visualization. WebSocket server on port 8765 streams real-time RL events. Gunicorn production WSGI server with Nginx reverse proxy.

Flask WebSocket Gunicorn
📊

Monitoring Stack

Prometheus + Grafana

Custom Prometheus exporter scrapes Redis metrics (queue depth, TPS history, reward stats). Pre-configured Grafana dashboard with auto-provisioning. Data retained for 7 days. Grafana on port 3000.

Prometheus Grafana
🦙

Ollama Integration

omnitech-ollama / FastAPI

Separate Ollama deployment stack with a FastAPI chat bridge. Serves LLM inference via local Ollama on port 11434. API layer on port 9000 exposes /chat endpoint for direct model interaction.

Ollama FastAPI llama3
💾

Redis Message Bus

redis:7-alpine

Central message bus and state store. Queues: jobs, results, history, security_alerts. Keys: scheduler_ready, queue_depth, worker_heartbeat, scheduler_heartbeat. Persistent with save 60 1.

📐 Reward Function
# Composite reward — maximize speed & accuracy, penalize model size def compute_reward(tps, acc, size_mb): reward = ( tps * W_TPS # speed: 0.6 × tokens/sec + acc * W_ACC # accuracy: 30.0 × eval score (0-1) - size_mb * W_SIZE# size: -0.01 × model size in MB ) return reward # UCB1 selection formula ucb = { q: scores[q] / counts[q] + math.sqrt(2 * math.log(total_steps) / counts[q]) for q in QUANTS } best_quant = max(ucb, key=ucb.get) # EMA score update (α=0.1) scores[q] = (1 - alpha) * scores[q] + alpha * new_reward

🚀 Deployment Guide

OPTION 1 — Docker (Recommended)

1

Clone & enter directory

cd omnitech-omega-v4

2

Add your model

Copy your f16 GGUF model: ./models/mistral-7b.f16.gguf

3

Launch all services

docker-compose up -d

4

Access dashboards

Dashboard: http://localhost:5000
Grafana: http://localhost:3000

OPTION 2 — Bare Metal

1

Install Python dependencies

pip install -r requirements.txt

2

Build llama.cpp

Clone and build llama.cpp, place binary at ./llama.cpp/quantize

3

Start Redis

redis-server --daemonize yes

4

Run OMNITECH

chmod +x run.sh && ./run.sh

OPTION 3 — Bootable ISO

1

Build the ISO

chmod +x build_iso.sh && ./build_iso.sh

2

Flash to USB

dd if=omnitech-omega.iso of=/dev/sdX bs=4M status=progress
